Detailed Description

template<typename Thread_local_state_t>
struct flow::util::Thread_local_state_registry< Thread_local_state_t >::Tsp_wrapper

Simply wraps a boost::thread_specific_ptr<Tl_context>, adding absolutely no data or algorithms, purely to work-around a combination of (some?) clang versions and (some?) GNU STL impls giving a bogus compile error, when one tries optional<Thread_local_state_registry>.

The error is "the parameter for this explicitly-defaulted copy constructor is const, but a member or base requires it to be non-const" and references impl details of STL's optional. The solution is to wrap the thing in a thing that is itself already noncopyable in the proper way, unlike thread_specific_ptr (which, at least as of Boost-1.87, still has a copy-forbidding ctor/assigner that takes non-const ref).
